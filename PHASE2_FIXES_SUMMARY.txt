WHAT WAS FIXED
───────────────────────────────────────────────────────────────────────────────

1. SHARED SECRET STORAGE
   Issue: Secrets generated during activation were never stored in database
   Fix:
     - Added encrypted storage columns to `installs` table
     - Implemented AES-256-GCM encryption (authority/app/crypto.py)
     - Heartbeat endpoint decrypts and validates HMAC signatures
     - Database migration: 002_phase2_fixes.py

2. HMAC SIGNATURE VALIDATION
   Issue: Heartbeat endpoint accepted requests without validating signatures
   Fix:
     - Implemented full HMAC-SHA256 signature validation
     - Requests rejected if signature doesn't match
     - Happens before any license checks

3. REQUEST REPLAY PREVENTION
   Issue: Same request could be replayed within heartbeat window
   Fix:
     - Compute SHA256 hash of request payload
     - Check for identical payloads within 15-minute window
     - Reject replays with "replay_attack_detected" status

4. SDK CACHE MANAGEMENT
   Issue: SDKs stored token in cache but secret came from environment variables
   Fix:
     - Updated TokenCache to store token + secret as JSON
     - Both PHP and Node SDKs read secrets from cache
     - Backwards compatible with old plain-text JWT format
     - Secrets automatically updated after each heartbeat
